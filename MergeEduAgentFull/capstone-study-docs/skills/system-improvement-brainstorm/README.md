# 시스템 개선 브레인스토밍

## 용도

이 스킬은 `MergeEduAgent` 같은 기존 시스템을 한 단계 더 고도화하기 위한 아이디어를 체계적으로 모으고, 평가하고, 압축할 때 사용한다.

단순 브레인스토밍이 아니라 아래를 함께 다룬다.

- 멀티 에이전트 구조 개선
- learner memory / learner model 고도화
- pedagogy policy 분리
- verifier, eval, safety 체계 추가
- 수학적 로직, 알고리즘, 연구 실험 설계

## 대략적인 동작 방식

1. 먼저 사용자에게 실행 설정을 확인한다.
2. 아이디어 단계와 평가 단계 설정을 정한다.
3. 현재 시스템 문서와 코드, 기존 로드맵을 읽는다.
4. 여러 관점에서 아이디어를 생성하고, 내부적으로만 정리하고 압축한다.
5. 마지막에는 살아남은 아이디어만 쉬운 설명으로 다시 쓴다.

## 최종 산출물 성격

- 최종 영구 산출물은 `아이디어 설명용 md 파일 1개`가 원칙이다.
- 중간 과정, 라운드별 메모, 평가 로그는 기본적으로 최종 문서에 넣지 않는다.
- 각 아이디어는 `카테고리`, `핵심 아이디어`, `핵심 내용`, `무엇이 추가되거나 바뀌는지`, `기대 효과`, `가장 작은 다음 실험` 중심으로 정리한다.

즉, 이 스킬은 내부적으로는 다회전 브레인스토밍을 하더라도, 사용자가 받는 결과물은 "과정 설명"보다 "새로운 아이디어 자체"를 빠르게 이해할 수 있게 만드는 데 초점이 있다.

추천 preset은 `아이디어 5에이전트 x 10라운드`, `평가 5에이전트 x 3라운드`, `최종 5개`다.
다만 이 값은 자동 적용되지 않고, 사용자가 명시하거나 추천값 사용을 승인했을 때만 시작한다.
