# Ver1 실패 기록: 강의 자료 PDF 업로드 실패

## 테스트 환경

- 날짜: 2026-04-26
- 브라우저: Comet
- 실행 URL: `http://localhost:5190`
- 테스트 강의실: `Ver1 Browser Test Classroom`
- 교사 계정: `ver1.teacher@example.com`
- 학생 계정: `ver1.student1@example.com` 외 4개
- 업로드 대상 파일: `/Users/jhkim/Downloads/6주차 학습자료.pdf`

## 사전 통과한 항목

- 교사 계정 회원가입 및 로그인 정상 동작
- 학생 계정 5개 회원가입 정상 동작
- 학생별 4자리 코드 표시 정상 동작
- 교사 계정에서 학생 5명 초대 정상 동작
- 초대받은 학생 계정으로 로그인했을 때 해당 강의실 접근 정상 동작
- 학생 계정에서는 강의실 생성, 자료 추가 같은 교사 전용 조작이 노출되지 않음

## 실패 시나리오

- 시나리오: 교사가 강의실 안에서 `1주차`를 만든 뒤 `6주차 학습자료.pdf`를 세부 강의로 업로드한다.
- 기대 동작:
  - PDF 선택 후 `업로드` 버튼을 누르면 자료가 서버에 저장된다.
  - 업로드가 완료되면 `세부 강의` 목록에 `6주차 학습자료`가 표시된다.
  - 초대받은 학생이 다시 로그인하면 해당 강의 자료를 볼 수 있다.
  - 이후 학생이 자료 안에서 페이지 설명과 시험 보기 기능을 사용할 수 있다.
- 실제 동작:
  - 파일 선택까지는 정상 동작했다.
  - `업로드` 버튼을 누른 뒤 업로드가 실패했다.
  - 화면에 `Gemini PDF upload failed: AI upload_pdf 실패: Gemini upload failed: 400 INVALID_ARGUMENT. API Key not found. Please pass a valid API key.` 오류가 표시됐다.
  - 서버 로그에서는 `/bridge/upload_pdf` 요청이 `500 Internal Server Error`로 기록됐다.
  - 코드 확인 결과 AI 브리지는 `.env`의 `GOOGLE_API_KEY`를 읽도록 되어 있으나, Gemini API가 현재 키를 유효한 키로 인정하지 않는 상태로 보인다.

## 재현 절차

1. 교사 계정 `ver1.teacher@example.com`으로 로그인한다.
2. `Ver1 Browser Test Classroom` 강의실에 들어간다.
3. `1주차`에서 `세부 강의 추가`를 누른다.
4. 강의 이름을 `6주차 학습자료`로 입력한다.
5. 파일 선택창에서 `/Users/jhkim/Downloads/6주차 학습자료.pdf`를 선택한다.
6. `업로드`를 누른다.

## 영향

- 강의 자료가 생성되지 않으므로 학생 계정에서 자료 노출 여부를 확인할 수 없다.
- 페이지 설명, 시험 보기 기능은 업로드 실패 때문에 이어서 검증할 수 없다.
- 현재 상태에서는 Gemini API 키가 없거나 유효하지 않을 때 사용자에게 복구 가능한 대안이 제공되지 않는다.

## 개선 필요

- 실행 환경에서 Gemini API 키 설정 여부와 유효성을 업로드 전에 검증해야 한다.
- API 키가 없는 개발 환경에서는 PDF 원본 저장만 먼저 허용하거나, 명확한 설정 안내를 제공하는 실패 상태가 필요하다.
- 업로드 실패 시 모달에 긴 원문 오류를 그대로 노출하기보다, 사용자가 이해할 수 있는 짧은 메시지와 관리자 확인용 상세 로그를 분리하는 것이 좋다.
