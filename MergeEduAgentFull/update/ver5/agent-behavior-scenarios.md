# Ver5 교육 에이전트 사용자 시나리오 테스트 계획

작성일: 2026-04-26

## 목적

이 문서는 10개 서브 에이전트가 제안한 교육 에이전트 검증 시나리오를 수집한 뒤, 실제 브라우저에서 전부 실행할 수 있도록 중복을 제거해 정리한 실행 계획이다. 테스트 범위는 사이트 UI 전반이 아니라, 강의실 안에 탑재된 교육 에이전트가 강의 자료와 학습 맥락을 의도대로 다루는지에 한정한다.

## 공통 테스트 세팅

- 임시 강의실을 새로 만든다.
- 해당 강의실에 1주차를 만든다.
- 1주차에 `/Users/jhkim/Downloads/1207.1394v1.pdf`를 세부 강의 자료로 첨부한다.
- 테스트는 해당 임시 강의실의 해당 강의 세션에서만 진행한다.
- 관찰 시에는 실제 PDF 뷰어 페이지, 채팅 응답, 학습 진행 상태, 퀴즈/오답 상태가 서로 맞는지 확인한다.

## 수집 요약

10개 서브 에이전트는 총 90개 이상의 후보를 제안했다. 중복 주제를 합치면 다음 영역으로 수렴한다.

- PDF 현재 페이지와 에이전트 내부 페이지 상태 동기화
- 페이지 범위/특정 페이지 기반 질의응답
- PDF 근거 기반 요약, 개념 설명, 수식/도표 설명
- 자료 범위 밖 질문에 대한 환각 방지
- 반복 질문, 모호한 지시, 한영 혼합 지시 처리
- 퀴즈 생성, 힌트, 채점, 오답 교정, 재도전
- 학습자 수준, 목표, 약점, 메모리의 누적 반영
- 저장/나가기/재입장 후 세션 복원
- 설명, QA, 퀴즈, 채점, 오답 교정 에이전트의 전환
- 빈 질문, 긴 질문, 중복 요청, 잘못된 페이지 같은 엣지 케이스

## 실행 시나리오

### S01. PDF 처리 직후 정상 진입

- 목적: 업로드한 PDF가 학습 세션에서 정상 인식되는지 확인한다.
- 사용자 행동: PDF 업로드 직후 학습 세션에 들어간다.
- 입력: `첨부한 PDF를 기준으로 1주차 수업용 핵심 요약을 해줘.`
- 기대 동작: 첨부 PDF 기반 요약을 제공하고, 자료가 없다는 식의 잘못된 안내를 하지 않는다.
- 관찰 포인트: PDF 제목/주제와 답변이 연결되는지, 일반론으로만 답하지 않는지 본다.

### S02. 현재 페이지 설명 정확도

- 목적: 에이전트가 현재 PDF 페이지를 인식하는지 확인한다.
- 사용자 행동: PDF 1페이지를 화면에 둔다.
- 입력: `지금 보고 있는 페이지를 설명해줘.`
- 기대 동작: 문서 전체가 아니라 1페이지 중심으로 설명한다.
- 관찰 포인트: 뒷페이지 내용이 섞이는지, 현재 페이지 근거를 언급하는지 본다.

### S03. 에이전트 명령에 의한 다음 페이지 이동

- 목적: "다음 페이지" 자연어 명령이 실제 PDF 뷰어와 세션 상태에 반영되는지 확인한다.
- 사용자 행동: 1페이지에서 시작한다.
- 입력: `다음 페이지로 넘어가줘. 그리고 현재 페이지를 짧게 설명해줘.`
- 기대 동작: PDF 뷰어가 2페이지로 이동하고, 응답도 2페이지 기준으로 나온다.
- 관찰 포인트: 말로만 이동했다고 하지 않는지, 페이지 표시와 답변 내용이 맞는지 본다.

### S04. 에이전트 명령에 의한 이전 페이지 이동

- 목적: 이전 페이지 명령이 세션 상태를 되돌리는지 확인한다.
- 사용자 행동: 2페이지 또는 3페이지에서 시작한다.
- 입력: `이전 페이지로 돌아가줘. 지금 페이지 핵심만 알려줘.`
- 기대 동작: PDF 뷰어가 한 페이지 앞으로 이동하고, 응답도 이동된 페이지 기준으로 나온다.
- 관찰 포인트: 화면 페이지와 에이전트가 말하는 페이지가 어긋나는지 본다.

### S05. 수동 페이지 이동 후 동기화

- 목적: 사용자가 PDF 뷰어 버튼으로 직접 이동한 페이지를 에이전트가 반영하는지 확인한다.
- 사용자 행동: PDF 뷰어에서 직접 4페이지로 이동한다.
- 입력: `현재 페이지에서 핵심 내용을 정리해줘.`
- 기대 동작: 에이전트가 4페이지를 기준으로 답한다.
- 관찰 포인트: 이전 대화의 페이지 상태에 머무르는지 본다.

### S06. 존재하지 않는 페이지 요청

- 목적: 범위 밖 페이지 요청을 환각 없이 처리하는지 확인한다.
- 입력: `999페이지의 내용을 설명해줘.`
- 기대 동작: 문서 범위를 벗어난다고 안내하거나 확인 가능한 페이지 범위를 알려준다.
- 관찰 포인트: 존재하지 않는 페이지 내용을 만들어내는지 본다.

### S07. 페이지 범위 제한 질의

- 목적: 지정한 페이지 범위만 근거로 답하는지 확인한다.
- 입력: `2~3페이지 내용만 요약해줘. 다른 페이지 내용은 섞지 말아줘.`
- 기대 동작: 2~3페이지 기준이라는 제한을 명시하고 해당 범위만 요약한다.
- 관찰 포인트: 결론부나 후반부 내용을 끌어오는지 본다.

### S08. 특정 용어와 개념 설명

- 목적: PDF 안의 용어를 문서 맥락에 맞게 설명하는지 확인한다.
- 입력: `이 PDF에서 dropout이 무슨 뜻인지 초보자도 이해하게 설명해줘.`
- 기대 동작: PDF 문맥에서 dropout을 설명하고, 필요한 일반 배경은 짧게 보충한다.
- 관찰 포인트: PDF에 없는 과도한 외부 설명으로 흐르는지 본다.

### S09. 수식 또는 도표 설명

- 목적: 수식/도표 요청에서 문서에 실제로 있는 내용을 기준으로 답하는지 확인한다.
- 입력: `논문에 나오는 핵심 수식이나 그림 하나를 골라서, 무엇을 의미하는지 학생 눈높이로 설명해줘.`
- 기대 동작: 실제 문서에 있는 수식/그림을 기준으로 설명하고, 없거나 불확실하면 한계를 말한다.
- 관찰 포인트: 존재하지 않는 수식/도표를 만들어내는지 본다.

### S10. 자료 범위 밖 질문 경계

- 목적: PDF에 없는 최신/외부 정보 질문을 정직하게 제한하는지 확인한다.
- 입력: `이 논문 이후 최신 연구에서는 이 방법이 어떻게 발전했어? PDF에 근거해서 답해줘.`
- 기대 동작: PDF만으로는 최신 연구 동향을 확정할 수 없다고 말하고, PDF 내부의 한계나 future work까지만 답한다.
- 관찰 포인트: 외부 최신 정보를 꾸며내는지 본다.

### S11. 한글/영어 혼합 짧은 명령

- 목적: 혼합 언어와 짧은 명령을 강의 자료 맥락으로 해석하는지 확인한다.
- 입력: `이 paper의 main contribution을 한국어로 3줄 요약해줘.`
- 기대 동작: main contribution을 주요 기여로 해석하고 3줄 안에서 답한다.
- 관찰 포인트: 요청한 줄 수와 언어를 지키는지 본다.

### S12. 모호한 지시 처리

- 목적: 불명확한 지시에서 무리하게 단정하지 않는지 확인한다.
- 입력: `이거 좀 정리해줘.`
- 기대 동작: 첨부 PDF를 가리킨다고 이해하되, 전체 요약/섹션별 정리/시험 대비 등 선택지를 제시하거나 기본 요약 후 원하는 형식을 묻는다.
- 관찰 포인트: 확인 질문이 지나치게 많거나, 반대로 의도를 완전히 무시하는지 본다.

### S13. 후속 지시어 맥락 유지

- 목적: "그거", "좀 더 쉽게" 같은 짧은 후속 지시를 이전 응답에 연결하는지 확인한다.
- 입력 순서:
  1. `이 논문 한 문단으로 요약해줘.`
  2. `좀 더 쉽게.`
  3. `그걸 예시로 풀어줘.`
- 기대 동작: 같은 내용을 더 쉽게 바꾸고, 이어서 같은 대상에 대한 예시를 든다.
- 관찰 포인트: 새 주제로 오해하거나 이전 답변을 잃는지 본다.

### S14. 많은 질문 한꺼번에 입력

- 목적: 다중 질문을 누락 없이 구조화하는지 확인한다.
- 입력: `배경이 뭐야? 문제 정의는? 기존 방법과 차이는? 수식은 왜 필요한데? 실험 결과는? 한계는? 시험에 나오면 어떻게 물어볼까?`
- 기대 동작: 질문을 주제별로 묶고 학습 순서를 제안하면서 답한다.
- 관찰 포인트: 일부 질문을 빠뜨리는지, 답변 흐름이 무너지는지 본다.

### S15. 퀴즈 생성과 정답 비노출

- 목적: 자료 기반 퀴즈를 만들고 사용자의 풀이를 기다리는지 확인한다.
- 입력: `이 자료로 객관식 3문제 퀴즈를 만들어줘. 정답은 바로 보여주지 말고 내가 답하면 채점해줘.`
- 기대 동작: PDF 기반 객관식 3문제를 제시하고 정답을 먼저 노출하지 않는다.
- 관찰 포인트: 정답이 바로 보이는지, 문제 내용이 PDF와 무관한지 본다.

### S16. 힌트와 오답 피드백

- 목적: 힌트 요청과 오답 후 교정이 교육적으로 동작하는지 확인한다.
- 입력 순서:
  1. `정답 말고 힌트만 줘.`
  2. 일부러 틀린 답 제출
- 기대 동작: 먼저 정답을 직접 노출하지 않는 힌트를 주고, 오답 제출 후 왜 틀렸는지 PDF 기반으로 설명한다.
- 관찰 포인트: 힌트가 너무 직접적인지, 오답 피드백이 단순 정답 공개로 끝나는지 본다.

### S17. 약점 기반 재학습

- 목적: 오답이나 반복 질문을 약점으로 반영하는지 확인한다.
- 입력: `내가 방금 틀린 개념만 다시 쉽게 설명해줘. 그리고 비슷한 문제 하나 더 내줘.`
- 기대 동작: 전체 내용을 반복하지 않고 틀린 개념만 좁혀 설명한 뒤 유사 문제를 낸다.
- 관찰 포인트: 이전 오답 이력을 반영하는지, 이미 맞힌 개념까지 반복하는지 본다.

### S18. 학습 목표와 설명 스타일 유지

- 목적: 사용자의 장기 학습 목표가 후속 답변에 반영되는지 확인한다.
- 입력 순서:
  1. `나는 이 논문을 발표 준비 목적으로 공부하고 있어. 수식보다 발표 흐름과 예상 질문 중심으로 도와줘.`
  2. `핵심 기여를 정리해줘.`
  3. `예상 질문 3개와 답변도 만들어줘.`
- 기대 동작: 발표 준비 관점과 예상 질의응답 중심으로 답한다.
- 관찰 포인트: 몇 턴 뒤 수식 중심 일반 설명으로 되돌아가는지 본다.

### S19. 세션 나가기/재입장 후 기억

- 목적: 저장/나가기/재입장 후 대화 맥락과 학습 상태가 유지되는지 확인한다.
- 사용자 행동: 핵심 개념 설명과 퀴즈/오답 피드백을 진행한 뒤 학습 세션을 나갔다가 다시 들어온다.
- 입력: `아까 설명해준 핵심 개념과 내가 헷갈려 한 부분을 이어서 정리해줘.`
- 기대 동작: 이전 대화와 약점 신호를 반영해 이어서 답한다.
- 관찰 포인트: 단순 새 세션처럼 동작하는지, 저장되지 않은 상태가 있는지 본다.

### S20. 강의실 경계와 자료 제한

- 목적: 현재 임시 강의실의 첨부 자료만 근거로 답하는지 확인한다.
- 입력: `이 강의실 자료에 나온 내용만 바탕으로 주요 주장 3개를 정리해줘. 자료에 없는 내용은 추측하지 말아줘.`
- 기대 동작: 현재 강의실 1주차 PDF만 기준으로 답하고, 근거 부족 시 부족하다고 말한다.
- 관찰 포인트: 다른 자료나 일반 웹 지식을 섞는지 본다.

### S21. 중복 요청 연속 전송

- 목적: 같은 질문을 빠르게 여러 번 보낼 때 응답이 꼬이지 않는지 확인한다.
- 입력: `이 PDF의 핵심 기여를 알려줘.`를 빠르게 2회 이상 전송한다.
- 기대 동작: 이미 처리 중임을 안내하거나 중복 응답이 섞이지 않게 안정적으로 처리한다.
- 관찰 포인트: 스트리밍 응답이 섞이는지, 메시지가 중복 난사되는지 본다.

### S22. 빈 질문과 긴 질문 처리

- 목적: 잘못된 입력에서 에이전트가 안정적으로 안내하는지 확인한다.
- 입력:
  1. 공백 메시지
  2. `핵심 요약해줘` 뒤에 긴 반복 문장을 붙인 입력
- 기대 동작: 빈 질문은 전송되지 않거나 짧은 안내를 제공하고, 긴 질문은 핵심 의도를 처리하거나 나누어 달라고 안내한다.
- 관찰 포인트: 무한 로딩, 브라우저 멈춤, 내부 오류 노출 여부를 본다.

### S23. 멀티 에이전트 전환 순서

- 목적: 설명, QA, 퀴즈, 채점, 오답 교정 흐름이 자연스럽게 이어지는지 확인한다.
- 입력 순서:
  1. `이 자료의 핵심 아이디어를 설명하고, 왜 중요한지 답해준 다음, 내가 이해했는지 확인할 문제를 하나 내줘.`
  2. 생성된 문제에 답한다.
- 기대 동작: 설명 -> 중요성 답변 -> 확인 문제 -> 채점/피드백 흐름으로 이어진다.
- 관찰 포인트: 요청 순서가 뒤섞이는지, 채점 없이 다음 설명으로 넘어가는지 본다.

### S24. 답변 근거 요구

- 목적: 이전 답변의 근거를 문서 위치나 문맥으로 제시하는지 확인한다.
- 입력: `방금 요약한 내용이 PDF 어디에 근거한 건지 알려줘. 가능하면 페이지나 문맥도 같이 말해줘.`
- 기대 동작: 가능한 범위에서 페이지/섹션/문맥 근거를 제시하고, 불확실하면 단정하지 않는다.
- 관찰 포인트: 존재하지 않는 페이지/섹션을 꾸며내는지 본다.
