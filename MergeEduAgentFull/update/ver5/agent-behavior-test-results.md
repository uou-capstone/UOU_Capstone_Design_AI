## 요약

초기 실제 브라우저 테스트에서는 교육 에이전트가 시작 단계에서 막혔다. Comet에서 학습 세션에 진입하면 PDF는 렌더링됐지만, 브라우저가 사용하는 `/api/session/:sessionId/event/stream` 응답이 `final` chunk 없이 비어 끝나면서 `강의 시작 이벤트가 실패했습니다.`가 표시됐다.

코드 확인 결과, 스트림 라우트가 `req.close`를 응답 스트림 중단으로 오인해 정상 POST 요청을 abort하고 있었다. 이를 `req.aborted`와 `res.close` 기반으로 수정한 뒤 다시 Comet에서 확인했으며, 세션 진입, 페이지 이동, 질문응답, 퀴즈 생성, 채점, 오답 교정, 저장/재입장 흐름은 정상 동작했다.

추가로 브라우저 테스트 중 다음 두 가지 퀴즈 UX 문제를 발견해 함께 수정했다.

- 객관식 오답 피드백이 `정답은 c3 입니다.`처럼 내부 choice id를 노출했다.
- 퀴즈 생성 중 모델의 JSON answer delta가 채팅 본문에 잠시 노출됐다.

두 문제 모두 서버 테스트와 Comet 재검증에서 보정됐다.

### O02. 스트리밍 이벤트 시작 실패

- 관련 시나리오: S02 이후 전체
- 상태: FIXED
- 최초 관찰:
  - 세션 진입 후 PDF 1페이지는 보이나 `SESSION_ENTERED` 스트림이 빈 응답으로 종료.
  - 비스트림 `/event` 경로는 정상.
- 원인:
  - `req.on("close")`를 abort 신호로 사용해 정상 POST body close를 연결 종료로 오인.

  - `req.aborted`와 `res.close`만 abort 신호로 사용.

  - 닫힌 response에는 progress/final/error chunk를 쓰지 않도록 writer guard 추가.
- 재검증:
  - Comet에서 세션 새로고침 후 초기 이벤트, 사용자 메시지, 페이지 이동, 퀴즈 이벤트가 모두 `final`까지 완료됐다.

### O03. 페이지 이동과 내부 페이지 상태

- 관련 시나리오: S03, S04, S05
- 상태: PASS
- 실제 관찰:
  - `다음 페이지로 넘어가줘...` 요청 후 PDF 뷰어가 `2 / 8 페이지`로 이동하고, 진행 상태가 `~2페이지까지 진행`으로 갱신됐다.
  - 답변도 2페이지의 `Value of Information Problem`, entropy, polytree 내용을 기준으로 나왔다.
  - `이전 페이지로 돌아가줘...` 요청 후 최종적으로 PDF 뷰어와 진행 상태가 1페이지로 돌아갔다.
  - PDF 뷰어의 `다음` 버튼으로 수동 이동한 뒤 `현재 페이지` 질문을 했을 때 2페이지 기준으로 답했다.
- 참고:
  - 이전 페이지 요청 직후 아주 짧게 답변 스트리밍이 먼저 보이고 PDF 페이지가 뒤따라 바뀌는 순간이 있었다. 최종 상태는 맞았지만, 느린 환경에서는 일시적 어긋남처럼 보일 수 있다.

### O05. 퀴즈 생성, 채점, 오답 교정

- 관련 시나리오: S15, S16, S17
- 상태: PASS with notes
- 실제 관찰:
  - 객관식 퀴즈 생성 요청 후 퀴즈 모달이 열리고 정답은 사전 노출되지 않았다.
  - 일부러 오답을 제출하면 점수와 문항별 정오가 표시되고, 기준 미달 시 오답 교정 흐름으로 이어졌다.
  - `polytree에서 왜 이렇게 어려운 문제가 되는지 잘 모르겠어.`라는 진단 답변을 넣으면 틀린 개념만 좁혀 다시 설명하고 재시험을 제안했다.
- 수정 전 문제:
  - 객관식 오답 피드백이 `c1`, `c3` 같은 내부 choice id를 노출했다.
  - 퀴즈 생성 중 raw JSON이 잠시 채팅에 노출됐다.
- 수정 후 재검증:
  - 오답 피드백은 실제 정답 선택지 문장으로 표시됐다.
  - 새 퀴즈 생성 중 JSON 객체는 채팅 본문에 노출되지 않았고, 최종 퀴즈 모달이 정상적으로 열렸다.
- 남은 UX 메모:
  - 사용자가 이미 `객관식`이라고 명시해도 오케스트레이터가 다시 퀴즈 유형 선택 버튼을 묻는다. 기능상 실패는 아니지만 흐름이 한 번 더 걸린다.
  - 오답 복구 흐름 중 `객관식 퀴즈를 한 번 더 만들어줘` 요청은 곧바로 새 퀴즈가 아니라 복구 흐름의 재확인/설명으로 흡수될 수 있다. `복구 질문은 종료하고...`라고 명시하면 새 퀴즈 생성으로 전환된다.

## 구현 후 확인한 수용 기준

- `/event/stream` 정상 요청은 `final` NDJSON chunk로 종료된다.
- `SESSION_ENTERED`, `USER_MESSAGE`, `SET_CURRENT_PAGE`, `QUIZ_TYPE_SELECTED`, `QUIZ_SUBMITTED` 계열 흐름이 Comet에서 완료된다.
- 닫힌 스트림에 늦은 chunk를 쓰지 않도록 guard가 있다.
- MCQ 오답 피드백은 내부 id 대신 실제 선택지 문장을 표시한다.
- 퀴즈 생성 중 모델 answer JSON delta는 채팅 본문에 표시하지 않는다.
