# QA Follow-up Update

- 시간: `2026-04-15 14:31:39`
- 범위: `질문응답(QA) 후속 질문 문맥 유지`

## 무엇이 바뀌었나

이제 QA는 `이전 QA 질문/답변만` 따로 기억합니다.

- 첫 질문이면: 이전 QA 메모리를 비우고 새로 시작
- 후속 질문이면: 이전 질문과 이전 QA 답변만 함께 넘겨서 답변
- 다음 페이지로 이동하거나 새 설명이 시작되면: QA 메모리 초기화

중요:

- 설명 본문 전체는 QA에 다시 넘기지 않음
- 세션 전체 대화도 QA에 다시 넘기지 않음
- 오직 `질문 -> QA 답변 -> 질문 -> QA 답변` 흐름만 이어짐

## 동작 예시

1. 설명이 끝남
2. 사용자가 질문 1을 함
3. QA가 답변 1을 함
4. 사용자가 질문 2를 함
5. QA는 질문 1 + 답변 1을 참고해서 답변 2를 함
6. 사용자가 질문 3을 함
7. QA는 질문 1, 2와 답변 1, 2를 참고해서 답변 3을 함

그 뒤:

- 사용자가 `다음 페이지`라고 하면 QA 메모리는 지워지고 다음 페이지로 감
- 사용자가 새 독립 질문을 시작한다고 오케스트레이터가 판단하면 QA 메모리를 비우고 새 질문 흐름으로 시작

## 내부적으로는

- 세션 상태에 `QA 전용 thread memory` 추가
- 오케스트레이터가 `START_NEW` 또는 `FOLLOW_UP` 모드로 QA 호출
- QA agent prompt에만 이전 QA 문맥 주입
- 설명/페이지 이동 시 QA thread reset
