### 4.3 API 테스트

#### 4.3.1 헬스 체크

```bash

```

**예상 응답:**
```json

  "redis": "connected"
```

```bash
    "exam_type": "Five_Choice",
```

- [ ] 프로필 캐싱이 작동하는가? (첫 요청: Cache Miss, 두 번째 요청: Cache Hit)

```bash
    "audience_level": "University Students"
  }'
```

```json
  "status": "accepted",
```

## 🔍 문제 해결

### Redis 연결 실패 시
- Redis 컨테이너가 실행 중인지 확인: `docker ps | grep redis`
- 환경 변수 확인: `REDIS_HOST`, `REDIS_PORT`
- 네트워크 확인: `docker network ls`

### API 엔드포인트가 보이지 않을 때
- 라우터 등록 확인: `app/main.py`의 `app.include_router()` 호출
- 서버 재시작: `docker-compose restart ai-service`

### 진행률이 업데이트되지 않을 때
- Redis 연결 확인: `/health` 엔드포인트에서 Redis 상태 확인
- Task ID 확인: 올바른 Task ID로 조회하는지 확인
- 로그 확인: `docker-compose logs ai-service`
