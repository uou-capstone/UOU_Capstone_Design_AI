## 에이전트 설계 원칙

### 1. 모듈화 원칙
- 각 에이전트는 독립적인 컴포넌트로 설계
- 명확한 입력/출력 인터페이스 정의
- 재사용 가능한 구조

### 2. 비동기 처리 원칙
- I/O 바운드 작업은 반드시 비동기 처리
- `asyncio`를 활용한 병렬 처리
- 블로킹 방지를 위한 `asyncio.to_thread` 활용

### 3. 에러 처리 원칙
- 각 단계별 명확한 에러 메시지
- Fallback 메커니즘 제공
- 사용자 친화적인 에러 응답

### 4. 성능 최적화 원칙
- Ordered Streaming: 결과를 순차적으로 스트리밍
- Prefetching: 사용자 대기 시간 활용
- Rate Limiting: API 호출 제한 관리

## 에이전트 추론 과정 시각화 및 스트리밍 가이드

에이전트가 최종 답변을 내기 전 내부적으로 수행하는 '생성적 추론(Generative Reasoning)' 과정을 사용자에게 실시간으로 보여주기 위한 설계 원칙입니다.

### 1. 데이터 구조 (Schema)

추론 과정은 최종 답변과 분리된 `thought` 필드를 가지며, 스트리밍 시 특수 토큰이나 이벤트를 통해 구분합니다.

#### Thought Node
각 추론 단계의 단위입니다.

```python
class ThoughtNode(BaseModel):
    step_id: str  # 단계별 고유 식별자
    content: str  # 실시간 스트리밍되는 텍스트 내용
    status: Literal["processing", "completed", "failed"]  # 상태
    visual_type: Optional[Literal["flowchart", "math_block", "graph"]]  # 시각화 타입
    timestamp: float  # 타임스탬프
```

#### 스트리밍 이벤트 타입

```python
class StreamingEvent(BaseModel):
    type: Literal["thought", "answer", "error"]  # 이벤트 타입
    delta: Optional[str] = None  # 증분 텍스트 (스트리밍용)
    content: Optional[ThoughtNode] = None  # 완전한 Thought Node
    metadata: Optional[Dict[str, Any]] = None  # 추가 메타데이터
```

### 2. 스트리밍 프로토콜 (Server-Sent Events)

백엔드에서 데이터를 보낼 때 답변(`answer`)과 생각(`thought`)의 채널을 분리합니다.

#### JSON 스트리밍 예시

```json
// 추론 과정 스트리밍
{ "type": "thought", "delta": "마르코프 프로세스의 시간 균일성(Time-homogeneity)을 검토 중..." }
{ "type": "thought", "delta": "\n전이 확률 P(X_{t+1}|X_t)가 t에 독립적인지 확인합니다." }
{ "type": "thought", "delta": "\n- 시간 균일성 가정이 계산 효율성에 미치는 영향 분석" }
{ "type": "thought", "delta": "\n- Edge case: 상태 공간이 변하는 동적 환경 검토" }

// 최종 답변 스트리밍
{ "type": "answer", "delta": "일반적인 마르코프 체인 모델에서는..." }
{ "type": "answer", "delta": " 시간 균일성을 가정합니다." }
```

#### 완전한 Thought Node 예시

```json
{
  "type": "thought",
  "content": {
    "step_id": "step_001",
    "content": "마르코프 프로세스의 시간 균일성(Time-homogeneity)을 검토 중...",
    "status": "completed",
    "visual_type": "math_block",
    "timestamp": 1705123456.789
  }
}
```

### 3. 시각화 컴포넌트 설계 (React/Vue 예시)

사용자 화면에서는 `thought` 타입의 데이터를 별도의 'Thinking Box' 또는 'Accordion' UI에 렌더링합니다.

### 4. 코드 리팩토링 예시 (Python/FastAPI 기반)

에이전트의 내부 로직에서 생각을 기록하고 내보내는 구조입니다.

#### 기본 스트리밍 제너레이터

```python
import asyncio
from typing import AsyncGenerator, Dict, Any
from pydantic import BaseModel

class ThoughtNode(BaseModel):
    step_id: str
    content: str
    status: str  # "processing" | "completed" | "failed"
    visual_type: str | None = None  # "flowchart" | "math_block" | "graph"
    timestamp: float

class StreamingEvent(BaseModel):
    type: str  # "thought" | "answer" | "error"
    delta: str | None = None
    content: ThoughtNode | None = None
    metadata: Dict[str, Any] | None = None

async def generate_agent_response(user_input: str) -> AsyncGenerator[StreamingEvent, None]:
    """
    에이전트 응답을 스트리밍으로 생성합니다.
    추론 과정(thought)과 최종 답변(answer)을 분리하여 전송합니다.
    """
    import time
    
    # 1. 초기 생각 시작
    thought_1 = ThoughtNode(
        step_id="step_001",
        content="사용자의 질문을 분석하고 관련 개념(RL, 확률론)을 인덱싱합니다.",
        status="processing",
        timestamp=time.time()
    )
    yield StreamingEvent(type="thought", content=thought_1)
    
    await asyncio.sleep(0.3)  # 실제 추론 로직 시뮬레이션
    
    thought_1.status = "completed"
    yield StreamingEvent(type="thought", content=thought_1)
    
    # 2. 중간 추론 (이미지 속 'Confirming Time Homogeneity' 단계)
    thought_process = [
        ("step_002", "시간 균일성 가정이 계산 효율성에 미치는 영향 분석", "math_block"),
        ("step_003", "Edge case: 상태 공간이 변하는 동적 환경 검토", "flowchart"),
    ]
    
    for step_id, step_content, visual_type in thought_process:
        thought = ThoughtNode(
            step_id=step_id,
            content=step_content,
            status="processing",
            visual_type=visual_type,
            timestamp=time.time()
        )
        yield StreamingEvent(type="thought", content=thought)
        
        await asyncio.sleep(0.5)  # 실제 추론 로직 시뮬레이션
        
        thought.status = "completed"
        yield StreamingEvent(type="thought", content=thought)
    
    # 3. 최종 답변 생성
    answer_parts = [
        "네, 대부분의 교과서적 마르코프 프로세스는 ",
        "계산의 편의를 위해 시간 균일성을 가정합니다. ",
        "이는 전이 확률이 시간에 독립적이라는 의미입니다."
    ]
    
    for part in answer_parts:
        yield StreamingEvent(type="answer", delta=part)
        await asyncio.sleep(0.1)  # 스트리밍 효과
```

#### FastAPI 엔드포인트 통합

```python
from fastapi import APIRouter
from fastapi.responses import StreamingResponse
import json

router = APIRouter(prefix="/api/agent", tags=["Agent"])

@router.post("/stream")
async def stream_agent_response(request: AgentRequest):
    """
    Server-Sent Events (SSE)를 사용하여 에이전트 응답을 스트리밍합니다.
    """
    async def event_generator():
        async for event in generate_agent_response(request.user_input):
            # SSE 형식으로 전송
            data = event.model_dump_json()
            yield f"data: {data}\n\n"
    
    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "Connection": "keep-alive",
            "X-Accel-Buffering": "no"  # Nginx 버퍼링 비활성화
        }
    )
```

#### Gemini API Thinking 기능 활용

```python
from google import genai
from google.genai import types

async def generate_with_thinking_stream(
    client: genai.Client,
    prompt: str
) -> AsyncGenerator[StreamingEvent, None]:
    """
    Gemini API의 thinking 기능을 활용하여 추론 과정을 스트리밍합니다.
    """
    # Gemini API 호출 (thinking 모드 활성화)
    response = await asyncio.to_thread(
        client.models.generate_content_stream,
        model="gemini-2.5-flash",
        contents=prompt,
        config=types.GenerateContentConfig(
            thinking_budget=10000,  # thinking 토큰 예산
            # response_mime_type="application/json"  # 필요시
        )
    )
    
    current_thought = ""
    current_answer = ""
    
    async for chunk in response:
        # Thinking 부분 처리
        if hasattr(chunk, 'candidates') and chunk.candidates:
            candidate = chunk.candidates[0]
            
            # Thinking content 추출
            if hasattr(candidate, 'content') and candidate.content:
                for part in candidate.content.parts:
                    # Thinking 부분 (내부 추론)
                    if hasattr(part, 'text') and part.text:
                        # Gemini의 thinking 출력을 파싱하여 thought로 변환
                        if "thinking" in part.text.lower() or "추론" in part.text:
                            current_thought += part.text
                            yield StreamingEvent(
                                type="thought",
                                delta=part.text
                            )
                        else:
                            # 최종 답변 부분
                            current_answer += part.text
                            yield StreamingEvent(
                                type="answer",
                                delta=part.text
                            )
```

### 6. 프론트엔드 통합 예시 (JavaScript)

```javascript
// EventSource를 사용한 SSE 수신
const eventSource = new EventSource('/api/agent/stream?user_input=' + encodeURIComponent(userInput));

const thinkingBox = document.getElementById('thinking-box');
const answerBox = document.getElementById('answer-box');

eventSource.onmessage = (event) => {
    const data = JSON.parse(event.data);
    
    if (data.type === 'thought') {
        // 추론 과정 UI에 추가
        const thoughtElement = document.createElement('div');
        thoughtElement.className = 'thought-item';
        thoughtElement.textContent = data.delta || data.content?.content;
        thinkingBox.appendChild(thoughtElement);
    } else if (data.type === 'answer') {
        // 답변 UI에 추가 (타이핑 효과)
        answerBox.textContent += data.delta;
    }
};

eventSource.onerror = (error) => {
    console.error('SSE 연결 오류:', error);
    eventSource.close();
};
```

### 7. 시각화 데이터 구조 및 렌더링 전략

#### 7.1 시각화 데이터 구조 설계

`thought` 이벤트 내부에 시각화 전용 필드를 추가하여, 프론트엔드의 Mermaid.js나 React Flow와 연동하기 위한 규격입니다.

##### ThoughtNode 확장 (Schema)

`streaming_schemas.py`의 `ThoughtNode`에 다음 필드를 포함합니다:

```python
class ThoughtNode(BaseModel):
    step_id: str
    content: str
    status: str = "processing"  # processing, completed, failed
    # 시각화를 위한 필드 추가
    viz_type: Optional[str] = None  # flowchart, graph, tree, sequence
    viz_data: Optional[str] = None  # Mermaid 문법 또는 JSON 좌표
    timestamp: float = time.time()
```

##### 실제 시각화 데이터 예시

에이전트가 **DQN(Deep Q-Network)**을 설명할 때, 내부적으로 생성하여 보낼 수 있는 시각화 데이터 예시입니다:

```json
{
  "type": "thought",
  "content": {
    "step_id": "step_02",
    "content": "DQN의 핵심인 Experience Replay와 Target Network의 관계를 시각화합니다.",
    "viz_type": "flowchart",
    "viz_data": "graph LR\n  A[Experience] --> B(Replay Buffer)\n  B --> C{Sampling}\n  C --> D[Main Network]\n  E[Target Network] -- Update --> D",
    "status": "processing"
  }
}
```

#### 7.2 프론트엔드 렌더링 전략

프론트엔드에서는 `viz_data`가 존재할 경우 이를 동적으로 차트로 변환합니다.

##### 시각화 컴포넌트 흐름

1. **데이터 수신**: SSE를 통해 `thought` 이벤트를 받습니다.
2. **파싱**: `viz_data`가 Mermaid 형식이면 `mermaid.render()`를 호출합니다.
3. **UI 업데이트**: 생각 상자(Thinking Box) 옆에 실시간으로 생성되는 다이아그램을 노출합니다.

##### React 컴포넌트 예시

```tsx
import { useEffect, useRef } from 'react';
import mermaid from 'mermaid';

interface ThoughtNode {
  step_id: string;
  content: string;
  status: 'processing' | 'completed' | 'failed';
  viz_type?: 'flowchart' | 'graph' | 'tree' | 'sequence';
  viz_data?: string;
}

function VisualizationBox({ thought }: { thought: ThoughtNode }) {
  const mermaidRef = useRef<HTMLDivElement>(null);
  
  useEffect(() => {
    if (thought.viz_data && thought.viz_type && mermaidRef.current) {
      // Mermaid 초기화
      mermaid.initialize({ 
        startOnLoad: true,
        theme: 'default',
        flowchart: { useMaxWidth: true }
      });
      
      // 고유 ID 생성
      const id = `mermaid-${thought.step_id}`;
      mermaidRef.current.id = id;
      
      // Mermaid 렌더링
      mermaid.render(id, thought.viz_data).then((result) => {
        if (mermaidRef.current) {
          mermaidRef.current.innerHTML = result.svg;
        }
      });
    }
  }, [thought.viz_data, thought.viz_type, thought.step_id]);
  
  if (!thought.viz_data) return null;
  
  return (
    <div className={`visualization-box ${thought.status === 'processing' ? 'pulsing' : ''}`}>
      <div ref={mermaidRef} className="mermaid-container"></div>
    </div>
  );
}

function ThinkingBox({ thoughts }: { thoughts: ThoughtNode[] }) {
  return (
    <div className="thinking-box">
      <h3>🤔 추론 과정</h3>
      {thoughts.map((thought) => (
        <div key={thought.step_id} className="thought-item">
          <div className="thought-content">
            <p>{thought.content}</p>
            {thought.viz_data && (
              <VisualizationBox thought={thought} />
            )}
          </div>
          <span className={`status-badge ${thought.status}`}>
```

#### 7.3 시각화 생성 원칙

에이전트가 시각화 데이터를 생성할 때 다음 원칙을 준수해야 합니다:

##### 1. 단계별 노출
- 모든 추론 단계를 한꺼번에 그리지 않고, `step_id` 순서에 따라 점진적으로 노드를 추가합니다.
- 각 단계마다 독립적인 `viz_data`를 생성하여 단계별로 시각화가 업데이트되도록 합니다.

##### 2. 수식 연동
- `content` 내의 LaTeX 수식(`$$Q(s, a)$$`)과 시각화 노드 내의 텍스트가 일치하도록 생성합니다.
- Mermaid 다이어그램 내에서도 수식을 포함할 수 있도록 합니다.

##### 3. 상태 표시
- `status`가 `processing`일 때는 노드 테두리에 애니메이션(Pulse)을 주고, `completed` 시 고정합니다.
- 프론트엔드에서 상태에 따라 시각적 피드백을 제공합니다.

```javascript
// EventSource 재연결 로직
class ReconnectingEventSource {
  constructor(url, options = {}) {
    this.url = url;
    this.options = options;
    this.reconnectDelay = options.reconnectDelay || 1000;
    this.maxReconnectDelay = options.maxReconnectDelay || 30000;
    this.reconnectAttempts = 0;
    this.eventSource = null;
    this.listeners = new Map();
  }
  
  connect() {
    this.eventSource = new EventSource(this.url);
    
    this.eventSource.onopen = () => {
      this.reconnectAttempts = 0;
      console.log('SSE 연결 성공');
    };
    
    this.eventSource.onerror = () => {
      this.eventSource.close();
      
      if (this.reconnectAttempts < 10) {
        const delay = Math.min(
          this.reconnectDelay * Math.pow(2, this.reconnectAttempts),
          this.maxReconnectDelay
        );
        
        setTimeout(() => {
          this.reconnectAttempts++;
          console.log(`재연결 시도 ${this.reconnectAttempts}...`);
          this.connect();
        }, delay);
      } else {
        console.error('최대 재연결 시도 횟수 초과');
      }
    };
    
    this.eventSource.onmessage = (event) => {
      const data = JSON.parse(event.data);
      
      if (data.type === 'error') {
        console.error('서버 에러:', data.delta);
        // 에러 UI 표시
      } else {
        // 정상 처리
        this.emit('message', data);
      }
    };
  }
  
  on(event, callback) {
    if (!this.listeners.has(event)) {
      this.listeners.set(event, []);
    }
    this.listeners.get(event).push(callback);
  }
  
  emit(event, data) {
    if (this.listeners.has(event)) {
      this.listeners.get(event).forEach(callback => callback(data));
    }
  }
  
  close() {
    if (this.eventSource) {
      this.eventSource.close();
    }
  }
}

// 사용 예시

  // 데이터 처리
});
```

#### 8.3 클라이언트 측 렌더링 최적화

스트리밍되는 텍스트를 실시간으로 Mermaid 차트로 그릴 때 화면이 깜빡거릴 수 있습니다.

##### Debounce 적용

```typescript
import { useEffect, useRef } from 'react';
import mermaid from 'mermaid';

function VisualizationBox({ thought }: { thought: ThoughtNode }) {
  const mermaidRef = useRef<HTMLDivElement>(null);
  const renderTimeoutRef = useRef<NodeJS.Timeout | null>(null);
  
  useEffect(() => {
    if (thought.viz_data && thought.viz_type && mermaidRef.current) {
      // Debounce: 300ms 대기 후 렌더링
      if (renderTimeoutRef.current) {
        clearTimeout(renderTimeoutRef.current);
      }
      
      renderTimeoutRef.current = setTimeout(() => {
        const id = `mermaid-${thought.step_id}`;
        mermaidRef.current!.id = id;
        
        mermaid.render(id, thought.viz_data!).then((result) => {
          if (mermaidRef.current) {
            mermaidRef.current.innerHTML = result.svg;
          }
        });
      }, 300); // 300ms debounce
    }
    
    return () => {
      if (renderTimeoutRef.current) {
        clearTimeout(renderTimeoutRef.current);
      }
    };
  }, [thought.viz_data, thought.viz_type, thought.step_id]);
  
  // ... 나머지 코드
}
```

##### LaTeX 호환성

마크다운 내의 수식(`$$..$$`)과 Mermaid 노드 내의 텍스트가 충돌하지 않도록 처리:

```typescript
import katex from 'katex';
import 'katex/dist/katex.min.css';

function ThoughtContent({ content, viz_data }: { content: string, viz_data?: string }) {
  // LaTeX 수식 렌더링
  const renderLatex = (text: string) => {
    // $$...$$ 패턴 찾기
    const latexRegex = /\$\$(.*?)\$\$/g;
    let lastIndex = 0;
    const parts: JSX.Element[] = [];
    let match;
    let key = 0;
    
    while ((match = latexRegex.exec(text)) !== null) {
      // 이전 텍스트
      if (match.index > lastIndex) {
        parts.push(<span key={key++}>{text.substring(lastIndex, match.index)}</span>);
      }
      
      // LaTeX 수식
      const latexHtml = katex.renderToString(match[1], { throwOnError: false });
      parts.push(<span key={key++} dangerouslySetInnerHTML={{ __html: latexHtml }} />);
      
      lastIndex = match.index + match[0].length;
    }
    
    // 남은 텍스트
    if (lastIndex < text.length) {
      parts.push(<span key={key++}>{text.substring(lastIndex)}</span>);
    }
    
    return parts.length > 0 ? parts : text;
  };
  
  return (
    <div className="thought-content">
      <div className="text-content">
        {renderLatex(content)}
      </div>
      {viz_data && (
        <VisualizationBox thought={{ viz_data, viz_type: 'flowchart' }} />
      )}
    </div>
  );
}
```

#### 9.4 캐싱 전략 가이드라인

##### 캐시 키 설계

- **명명 규칙**: `{타입}:{해시 또는 ID}`
  - 예: `profile:abc123def456`, `lecture:dqn_chapter_1`
- **해시 사용**: 긴 텍스트는 MD5 해시로 변환하여 키 생성
- **네임스페이스**: 타입별로 접두사 사용하여 충돌 방지

##### TTL (Time To Live) 설정

- **프로필**: 24시간 (86400초) - 강의 내용이 변경되지 않으면 재사용
- **강의 대본**: 7일 (604800초) - 챕터별로 캐싱
- **시험 문제**: 1시간 (3600초) - 문제는 자주 변경될 수 있음

##### Fallback 전략

Redis 연결 실패 시에도 서비스가 정상 동작하도록 Fallback 구현:

```python
try:
    cached_data = await redis.get(cache_key)
    if cached_data:
        return cached_data
except Exception as redis_error:
    # Redis 오류 시 직접 생성 (Fallback)
    print(f"[Redis] Error: {redis_error}. Falling back to direct generation.")
    return await generate_directly()
```

   ```json
     "topic": "Python 프로그래밍"
   }
   ```

##### 진행률 단계

- **0%**: 작업 대기 중 (queued)
- **10%**: 기획안 작성 중 (Phase 1)
- **30%**: 기획 검토 중 (Phase 2)
- **50%**: 본문 집필 중 (Phase 3, 병렬 처리)
- **80%**: 내용 검증 및 수정 중 (Phase 4)
- **90%**: 최종 문서 생성 중 (Phase 5)
- **100%**: 완료 (completed)

##### 프론트엔드 폴링 예시

```typescript
async function pollTaskStatus(taskId: string): Promise<any> {
  const response = await fetch(`/note-gen/status/${taskId}`);
  const status = await response.json();
  
  if (status.status === 'completed') {
    return status.result; // 생성된 마크다운 반환
  } else if (status.status === 'failed') {
    throw new Error(status.error);
  } else {
    // 진행 중이면 2초 후 다시 조회
    await new Promise(resolve => setTimeout(resolve, 2000));
    return pollTaskStatus(taskId);
  }
}

// 사용 예시
const { task_id } = await fetch('/note-gen/generate-async', {
  method: 'POST',
  body: JSON.stringify({ topic: 'Python', audience_level: 'University Students' })
}).then(r => r.json());

const result = await pollTaskStatus(task_id);
console.log('생성된 강의 노트:', result);
```
